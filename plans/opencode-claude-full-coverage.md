# Full Claude Code Coverage from OpenCode

## TL;DR

- Extends the existing `opencode-to-claude.mjs` converter so `src/claude/` is a **complete** generation of `src/opencode/` — closing the three previously-deferred gaps: the notification plugin, MCP servers, and `tui.jsonc`.
- **3 translation additions**: (1) port `plugins/notification.js` to Claude hooks + a generated stateful `notify.sh` runtime, (2) translate the 5 MCP servers into `.mcp.json` (all marked disabled), (3) explicitly document `tui.jsonc` as not portable.
- **10 tasks** total across converter code, a new shell template, settings schema changes, source-doc drift fixes, and ADR removal.
- Most critical / highest effort: the **stateful notification port** (Claude hooks are stateless shell commands; the rich body must be reconstructed from a per-session temp-state file).
- Estimated overall effort: **medium** (~1 focused session). Critical path is notify.sh template → hooks wiring → settings.json schema → verify in-sync.

## Overview

`src/opencode/` is the single source of truth; `src/claude/` is generated by `etc/scripts/src/ai/opencode-to-claude.mjs`. Today the converter covers `AGENTS.md → CLAUDE.md`, `agent/`, `command/`, `skills/`, and a minimal `settings.json` + empty `.mcp.json`. Three OpenCode pieces are **not** yet generated: `plugins/notification.js`, the `mcp` block of `opencode.jsonc`, and `tui.jsonc`. This spec makes the converter cover all of them so the Claude config is a faithful, fully-generated mirror.

Guiding constraint (the user's "separate script that handles all of the Claude Code stuff"): **all** Claude-config generation stays owned by the single `opencode-to-claude.mjs` converter — no Claude logic is scattered elsewhere. The notification *runtime* is itself a separate, source-controlled shell script (`notify.sh`) that the converter emits into the generated tree.

## Architecture

Layers touched:

- **Converter (`etc/scripts/src/ai/opencode-to-claude.mjs`)** — add three generators: `generateHooksScript`, MCP translation inside `generateSettings`, and an explicit skip for `tui.jsonc`. This is the only place Claude logic lives.
- **New shell template (`etc/scripts/src/ai/templates/claude-notify.sh`)** — the hand-maintained *source* of the notification runtime. It is **not** under `src/claude/` (which must never be hand-edited); the converter copies it to `src/claude/hooks/notify.sh` during generation. This keeps `src/claude/` fully generated while the script stays editable.
- **Generated output (`src/claude/`)** — gains `hooks/notify.sh`; `settings.json` gains a `hooks` block, a `disabledMcpjsonServers` array, and an `env` block of sound defaults; `.mcp.json` gains the 5 translated servers.
- **Symlink wiring** — `sync_links.sh` already maps `src/claude → ~/.claude` as a whole directory (line 28), so `hooks/notify.sh` ships to `~/.claude/hooks/notify.sh` automatically. Hooks reference it by absolute path `$HOME/.claude/hooks/notify.sh`.
- **Source docs** — `src/opencode/AGENTS.md` (regenerated into `CLAUDE.md`) currently mis-names the plugin and tui file; fix at the source. Per the recorded decision, the `architecture/` ADRs are deleted and any ADR references scrubbed.

How a generated notification fires at runtime:

```
Claude Code event ──> settings.json hook (Stop / Notification / PostToolUse)
        └─> $HOME/.claude/hooks/notify.sh  (reads hook JSON on stdin)
                └─> reads/writes $TMPDIR/claude-notify-<session_id>.json (state)
                        └─> afplay/paplay (sound) + osascript/notify-send (desktop notification)
```

## Data flow

Generation (build-time):

1. `opencode-to-claude.sh` → `opencode-to-claude.mjs` runs from the dotfiles root.
2. Existing generators run (`CLAUDE.md`, agents, commands, skills) unchanged.
3. `generateSettings` parses `opencode.jsonc`:
   - permission all-allow → `permissions.defaultMode: "bypassPermissions"` (unchanged).
   - **NEW**: iterate `config.mcp` → build Claude `mcpServers` map for `.mcp.json`; collect `enabled:false` names into `settings.json.disabledMcpjsonServers`.
   - **NEW**: emit a `hooks` block (Stop / Notification / PostToolUse) pointing at `$HOME/.claude/hooks/notify.sh`, plus an `env` block of sound defaults.
4. **NEW** `generateHooksScript`: copy `templates/claude-notify.sh` → `src/claude/hooks/notify.sh`, `chmod 0755`.
5. **NEW**: `tui.jsonc` is intentionally not read; log an info line stating it is skipped (not portable).
6. JSON summary on stdout gains `hooks` and `mcpServers` counts.

Notification (run-time): see the diagram above — stateless Claude hook invocations are made stateful by reading/merging a per-session temp JSON file keyed on `session_id`.

## Tasks

> All converter tasks (T2–T6) modify the same file (`opencode-to-claude.mjs`) and so are **sequential** with respect to each other (or done as one edit pass). The template (T1) and docs (T7–T9) are **parallel-safe**.

### T1 — Author the notification runtime template `etc/scripts/src/ai/templates/claude-notify.sh` *(large, parallel-safe)*

- New file. Hand-written shell port of `src/opencode/plugins/notification.js`.
- Reads the Claude hook JSON from stdin using `node -e` (no `jq` dependency; `node` is already required by the converter).
- Maintains per-session state at `${TMPDIR:-/tmp}/claude-notify-<session_id>.json`: `taskStartTime`, `toolCallCount`, `lastToolName`, `changedFiles[]`.
- Behavior by `hook_event_name`:
  - `PostToolUse`: increment `toolCallCount`, set `lastToolName` from `tool_name`; if `tool_name` is `Edit`/`Write`/`NotebookEdit`, add `tool_input.file_path` to `changedFiles`.
  - `Stop`: compute elapsed since `taskStartTime`, build the multi-line body (`project - elapsed`, `Last: X | N calls | M files`, last message), play idle sound, send "Task completed" notification, delete the state file.
  - `Notification`: play permission sound, send "Waiting for input" notification (Claude fires `Notification` on permission prompts / idle waits).
  - No error variant (dropped for v1 — Claude has no per-error hook).
- Sound + notify helpers mirror the plugin: macOS `afplay -v <vol> /System/Library/Sounds/<name>.aiff` + `osascript display notification` with title **"Claude Code"**; Linux `paplay` + `notify-send`. Wrap every external call so failures never break Claude.
- Env overrides: `CLAUDE_SOUND_IDLE` (Glass), `CLAUDE_SOUND_PERMISSION` (Ping), `CLAUDE_SOUND_VOLUME` (0.3). Follow repo shell conventions (`set -e` where safe, function-based, braces on all control flow).
- Reuse the plugin's `formatDuration`/`truncate`/`getSessionName` logic.

### T2 — Add `generateHooksScript()` to the converter *(small, sequential after T1)*

- New generator in `opencode-to-claude.mjs`: `await emptyDir(path.join(outDir, "hooks"))`, copy `templates/claude-notify.sh` → `outDir/hooks/notify.sh`, then `fs.chmod(dest, 0o755)`.
- Resolve the template relative to the converter file (`new URL("templates/claude-notify.sh", import.meta.url)` or `path.join(scriptDir, ...)`), independent of CWD.
- Push to `failed[]` on copy/chmod error. Return a count for the summary.
- Refuse-to-delete guard already protects `outDir`; `hooks/` is inside it.

### T3 — Translate MCP servers in `generateSettings()` *(medium, sequential)*

- Add `translateMcpServers(config.mcp)` producing Claude `.mcp.json` `mcpServers`:
  - `type: "local"` + `command: [c0, ...rest]` → `{ "type": "stdio", "command": c0, "args": rest }` (include `env` if present).
  - `type: "remote"` + `url` → `{ "type": "http", "url": url }` (OpenCode "remote" maps to Claude `http`; SSE ambiguity noted in Open Questions).
  - Unknown `type` or missing `command`/`url` → push to `failed[]` and skip that server.
- Collect every server whose `enabled === false` into a sorted `disabledMcpjsonServers` array.
- Write the full server map to `.mcp.json` (replacing the empty placeholder).

### T4 — Extend the generated `settings.json` schema *(medium, sequential)*

- Add `disabledMcpjsonServers: [...]` from T3 (omit the key if empty).
- Add the `hooks` block:
  ```json
  "hooks": {
    "PostToolUse": [{ "matcher": "*", "hooks": [{ "type": "command", "command": "$HOME/.claude/hooks/notify.sh" }] }],
    "Stop":        [{ "hooks": [{ "type": "command", "command": "$HOME/.claude/hooks/notify.sh" }] }],
    "Notification":[{ "hooks": [{ "type": "command", "command": "$HOME/.claude/hooks/notify.sh" }] }]
  }
  ```
- Add an `env` block with the sound defaults so the hook inherits them (`CLAUDE_SOUND_IDLE` etc.).
- Keep existing `$schema` + `permissions.defaultMode`.

### T5 — Explicitly skip `tui.jsonc` *(small, sequential)*

## API contracts

Converter functions (signatures other tasks depend on):

```js
// T2
async function generateHooksScript(scriptDir, outDir, failed): Promise<number>

// T3 — pure transform, no IO
function translateMcpServers(mcp /* opencode.jsonc mcp block */):
  { servers: Record<string, ClaudeMcpServer>, disabled: string[], errors: string[] }

// ClaudeMcpServer (stdio):  { type: "stdio", command: string, args: string[], env?: object }
// ClaudeMcpServer (http):   { type: "http", url: string }
```

stdout JSON summary (extended):

```json
{ "claudeMd": 1, "agents": N, "commands": N, "skills": N, "settings": 1,
  "hooks": 1, "mcpServers": 5, "failed": [] }
```

Generated `notify.sh` runtime contract: reads one hook-event JSON object on stdin (`hook_event_name`, `session_id`, `cwd`, and for `PostToolUse` `tool_name` + `tool_input`); writes nothing to stdout that Claude consumes; always exits 0 (never block the agent).

## Edge cases

- **Stateless hooks across concurrent sessions**: state file must be keyed on `session_id`; missing `session_id` → fall back to a single shared file or a hash of `cwd`.
- **Stale state files**: `Stop` deletes its own file; add age-based cleanup (e.g., remove files older than N hours) so crashed sessions don't leak temp files.
- **Missing `node`/`jq` at hook time**: prefer `node` (already a hard converter dep) for JSON parse; degrade gracefully (still play sound, skip rich body) if absent.
- **Sound assets absent on Linux**: `paplay` of a fixed freedesktop sound may not exist; wrap and ignore, mirroring the plugin.
- **MCP `remote` → http vs sse**: OpenCode only says "remote"; defaulting to `http` may be wrong for SSE-only servers (see Open Questions).
- **MCP server missing `command`/`url` or unknown `type`**: skip + record in `failed[]` rather than emitting an invalid `.mcp.json`.
- **All MCP servers disabled (today's state)**: `.mcp.json` lists all 5 but `disabledMcpjsonServers` also lists all 5 → none auto-activate; verify Claude tolerates a server appearing in both.
- **`emptyDir` on `hooks/`**: ensure it only wipes the generated `hooks/` dir, never the template source.
- **Idempotency**: regeneration must produce byte-identical output (stable key ordering in `.mcp.json` and `settings.json`).
- **`bypassPermissions` + Notification hook**: with permissions bypassed, Claude may rarely fire `Notification`; the permission sound path may seldom trigger — acceptable, document it.

## Testing approach

- **Converter unit-level**: run the converter against `src/opencode` into a temp out-dir; assert `.mcp.json` has 5 servers with correct `stdio`/`http` shapes, `settings.json` has the `hooks` + `disabledMcpjsonServers` + `env` keys, and `hooks/notify.sh` exists and is executable.
- **Idempotency**: run twice into two temp dirs; `diff -r` must be empty.
- **`notify.sh` smoke**: pipe synthetic hook JSON for `PostToolUse` then `Stop`; assert the state file is created, accumulates counts, then is removed on `Stop`, and the script exits 0 even when `afplay`/`osascript` are stubbed/absent.
- **Negative MCP**: feed a malformed mcp entry; assert it lands in `failed[]` and the converter exits non-zero.
- Keep checks runnable via the existing `verify-all.sh`/lint conventions; no new test framework introduced.

## Decisions

Resolved during clarification (no open questions remain):

### Requirements
- **Error notifications**: drop the error variant in v1. Claude has no clean per-error hook; `notify.sh` handles only `Stop` (idle) and `Notification` (permission) sounds.
- **Notification title**: generated notifications use the title **"Claude Code"** (not "OpenCode").

### Architecture
- **Template parser**: parse hook stdin JSON with **`node -e`** (no new dependency; `node` is already required by the converter).
- **MCP transport for `remote`**: map all OpenCode `remote` servers to Claude **`http`**; document the assumption in code comments.
- **Template location**: editable source lives at **`etc/scripts/src/ai/templates/claude-notify.sh`**, copied into the generated tree by the converter.

### Scope
- **Doc-drift fixes (T7/T8)**: **in scope** for this change.
- **ADRs**: **delete all `architecture/` ADRs** and scrub references (T9). No new ADR is created.

### Conventions
- **Sound env-var naming**: use **`CLAUDE_SOUND_*`** (`CLAUDE_SOUND_IDLE` = Glass, `CLAUDE_SOUND_PERMISSION` = Ping, `CLAUDE_SOUND_VOLUME` = 0.3).

### Risks (still active — engineering constraints, not unresolved questions)
- Hooks run on **every** `PostToolUse` — keep the script fast and side-effect-light to avoid slowing the agent loop.
- A buggy `notify.sh` that exits non-zero could interfere with Claude's flow — it must **always** exit 0.
- Listing a server in both `.mcp.json` and `disabledMcpjsonServers` is the intended "available but off" pattern; if a future Claude version rejects that combination, the MCP approach needs revisiting.
